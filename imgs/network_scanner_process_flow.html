ARP Network Scanner — End-to-End Process Flow

How a Python + scapy ARP scanner broadcasts a request to a whole subnet and only the owning host replies

Prerequisites

  • • scapy installed in the Python environment
  • • Script run with root / admin privileges
  • • Attached to the same LAN as the targets

Inputs → Outputs

  • • Input: IP range, e.g. 192.168.1.0/24
  • • Output: list of (IP, MAC) pairs on the LAN
  • • Note: non-owning hosts never send a "not mine" reply — ARP has no negative response, they just stay silent

Tools & Integrations

  • • scapy (ARP / Ether / srp)
  • • Raw sockets via the OS network stack
  • • Local Ethernet / Wi-Fi broadcast domain
